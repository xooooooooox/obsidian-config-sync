# 1.0.0 documentation pass: README refresh + ARCHITECTURE.md

Two documentation deliverables marking the 1.0.0 stable milestone. The README is user-facing
and ~6 versions stale; there is no code-architecture doc anywhere in the repo. This closes both
gaps. No product/code changes — docs and screenshots only.

## Deliverable A — README refresh (`README.md` + `README.zh.md`)

The README's foundation is current (Sync Center, capture/apply, availability sections + install
engine, remotes/pull-push, sync modes/encryption, store layout, walkthroughs, security). It was
last edited 2026-07-13 (pre-0.28), so it misses six versions of features and carries a
self-contradicting screenshot. Changes, minimal and in place:

1. **Run history** (0.30) — add to the "How it works" result-strip paragraph: each run is also
   recorded in a browsable, clearable **History** (sidebar entry → table → per-run detail),
   with an honest tone (green/amber/red) and failures expanded by default.
2. **Cold-start Adopt** (0.33) — add to the transport/cold-start text: a fresh device discovers
   the store on its own and shows an **Adopt** banner + one-time guidance (apply the store, don't
   capture over it), independent of manually adding a remote.
3. **Stop syncing / leftover cleanup** (0.31) — add a short bullet: an item can be removed from
   sync (optionally deleting its store copy), and store files with no matching item surface as
   **Leftover** for cleanup.
4. **Version-pinned install** (0.29) — correct "fetch the plugin's latest release" to: installs
   are **pinned to the version the store recorded** (via the lock), falling back to latest-stable
   with a warning when that release is missing.
5. **Batch resilience + sticky strip** (0.35) — note that bulk install/apply **survives per-item
   failures** (one bad plugin becomes a `✗` row, the rest continue) and that the result strip is
   **pinned (sticky)** so the outcome stays visible while scrolling a long list.
6. **Screenshots** — regenerate from the current build in the dev vault: replace `sync-panel.png`
   and `apply-picker.png`; **drop `status-modal.png`** (the popup dialog it shows was retired —
   the text already says "rather than a popup dialog"). Keep `settings-picker.png` if still
   representative, else retake. Regeneration done via obsidian-cli against the dev vault (current
   1.0.0 build already installed there).
7. **Releasing section** — leave the attestation wording as-is (the flow is accurate); no change
   needed beyond confirming steps still match `version-bump.mjs` + CI.
8. **`README.zh.md`** — mirror every change so the bilingual pair stays in lockstep.

Scope guard: do **not** rewrite sections that are already accurate; edit only the lines the six
items touch. Preserve the existing voice and structure.

## Deliverable B — `docs/ARCHITECTURE.md` (new)

A maintainer/future-you orientation to how the code is built — the one gap the existing docs
(`CLAUDE.md`/`AGENTS.md` = agent instructions, `DESIGN.md` = UI design system, `specs/` = per-
feature history, private memory = running log) don't fill. Evergreen name (`ARCHITECTURE.md`,
not `handoff.md`); `CLAUDE.md`/`AGENTS.md` gain a one-line pointer to it.

Sections:

1. **Overview** — the three-layer shape and why: pure `src/core/*` (portable logic, no Obsidian
   imports) ⇄ `src/main.ts` (the only connector: wires Obsidian's `app.vault.adapter`,
   `app.plugins`, `requestUrl`, timers into `CoreContext`/`PluginHost`) ⇄ `src/ui/*` (views,
   modals, panel model). One data-flow line each for capture / apply / pull / push.

2. **Module map** — one sentence per file, grouped:
   - Engine & context: `ConfigSyncCore.ts` (capture/apply/applyWithActions/captureWithActions/
     revert/import/pushExternal + `CoreContext`/`PluginHost`), `types.ts`.
   - Status & availability: `status.ts`, `availability.ts`, `catalog.ts`, `pluginState.ts`,
     `leftover.ts`, `runHistory.ts`.
   - Transforms & storage: `modes.ts`, `crypto.ts`, `switchList.ts`, `pathing.ts`, `io.ts`,
     `sanitize.ts`, `manifest.ts`, `merge.ts`.
   - Install & discovery: `installer.ts`, `bratIndex.ts`, `pkm.ts`, `async.ts`.
   - UI: `SyncCenterView.ts` (the hub), `SettingTab.ts`, `panelModel.ts`, `reportContent.ts`,
     `diffView.ts`, `jsonView.ts`, `sensitiveSort.ts`, `commitGroups.ts`, `*Modal.ts`.
   - Connector: `main.ts`.

3. **Core invariants** — the rules a change must not break:
   - Pure core / connectors-are-the-only-classes; core takes injected `FileIO`/`PluginHost`.
   - Switch lists identified by group **name** (`SWITCH_LIST_GROUPS`) and compared as **sets**;
     the five alignment points (statusForGroups / classifyMerge / diffRemote / captureGroup /
     applyGroup) must stay consistent.
   - Enabled truth = **loaded OR persisted** (`pluginRuntimeEnabled`); `enabledPlugins` alone
     misclassifies a running-but-unpersisted plugin.
   - **Self-apply never disables/reloads config-sync** (`applyGroup` skips `config-sync`).
   - Backup/lock model: apply backs every touched file into `config-sync-backup` (outside the
     plugin folder), one-slot **Revert**; `store.lock.json` records source versions.
   - **configDir-agnostic** store (literal `configdir` segment) so `.obsidian` / `.obsidian_apple`
     both map.
   - Run history is a **separate local-only file**, never synced.
   - Bulk apply/install is **per-item isolated**; installs use timeout + retry.

4. **Data model** — `config-sync.json` (groups; schema-validated), `store.lock.json`
   (capturedAt + per-group `sourcePluginVersion`/`sourceAppVersion`), `store/` layout
   (`configdir/…` mirror + dotless vault-root files), `run-history.json` (local-only), and the
   plugin's own `data.json` (persisted groups, `bratPluginIndex`, PKM mode, history config).

5. **How to extend** — the concrete touch-points for: a new **group type** (`pathing` +
   capture/applyGroup), a new **sync mode** (`modes.ts`), a new **remote type** (`types.ts`
   `Remote` union + `main.ts` transport + `status.ts` check), a new **external store target**
   (`ExternalStoreReader`/`ExternalStoreWriter` in `ConfigSyncCore.ts`).

6. **Testing & gates** — vitest over pure core (27 files); lint **67-warning baseline** (two
   BRAT sentence-case false positives kept, no eslint-disable), `scripts/check-no-hardcoded-
   color.sh`, all CSS via theme vars + `body.is-mobile`/`is-phone` scoping; live dev-vault via
   **obsidian-cli (routes by CWD — run from `dev/vault/`)**; `npm run build` = tsc + esbuild.

7. **Current state & how to resume** (kept minimal — the only part that can rot):
   - 1.0.0 = first stable; 0.x history retained on GitHub.
   - Parked backlog (three items): audit polish (`DESIGN.md` §5, 6 findings), #5 capture/pull
     interruption robustness, history file diffs (idea1) — all deferred by the maintainer.
   - Release flow pointer: `npm version` → push tag → CI draft → hand-written notes → publish.
